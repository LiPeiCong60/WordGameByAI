# 公网部署准备清单

这个项目已经具备基础注册、登录、验证码、管理员后台和存档归属权限。上线公网前，还需要把下面这些生产环境能力补齐或确认。

## 必做项

- 域名和 HTTPS：准备域名，用 Nginx / Caddy / Traefik 做反向代理，并启用 TLS 证书。
- CORS：把 `CORS_ALLOW_ORIGINS` 改成你的正式前端域名，例如 `https://game.example.com`。
- API 地址：前端构建时设置 `VITE_API_BASE_URL=https://api.example.com/api`，不要继续用本地地址。
- 数据库：SQLite 只适合单机验证；公网多用户建议使用 MySQL，并配置自动备份。
- 密钥：只在服务器环境变量或私有 `.env` 中保存 `OPENAI_API_KEY`，不要提交仓库。
- 管理员账号：在服务器设置长随机 `ADMIN_BOOTSTRAP_TOKEN`，创建首位管理员时填写该密钥；确认成功后轮换或删除，且保持 `ALLOW_FIRST_USER_ADMIN=false`。
- 模板权限：管理员创建公共模板；普通用户创建私人模板，只自己和管理员可见。
- 消息额度：普通非会员默认每日 20 条；管理员可在后台设置会员和每个用户的每日上限。
- 进程守护：用 systemd、Docker Compose、Supervisor 或 PM2 类工具托管后端和前端静态服务。
- 反向代理流式配置：剧情接口使用流式响应，Nginx 需要关闭代理缓冲，避免流式输出被攒成一次性返回。
- 上传目录：`backend/uploads` 需要持久化挂载，并加入备份策略。
- 防火墙：只开放 80、443 和必要 SSH 端口，后端服务端口不要直接暴露公网。

## 强烈建议

- 验证码升级：当前是轻量数学 SVG 验证码，适合本地或小范围测试；公网建议接入 Cloudflare Turnstile、hCaptcha 或 reCAPTCHA。
- 限流：项目已内置用户级消息短窗限流；公网开放注册时仍建议给 `/api/auth/register`、`/api/auth/login`、`/api/auth/captcha` 增加反向代理层 IP 限流。
- 审计日志：记录登录、注册、管理员操作、存档导入导出和失败的权限访问。
- Token 策略：根据场景调整 `AUTH_TOKEN_TTL_DAYS`，并补登出/撤销全部 token 的管理能力。
- 数据迁移：当前使用启动时轻量迁移，正式生产建议引入 Alembic 之类的迁移工具。
- 内容和费用控制：给大模型调用增加每日额度、并发限制、超时和失败重试策略。
- 备份恢复演练：定期验证数据库和上传目录能从备份恢复。
- 监控告警：至少监控进程存活、接口错误率、磁盘空间、模型调用失败率和费用异常。

## 推荐部署形态

```text
用户浏览器
  -> HTTPS 域名
  -> Nginx / Caddy
  -> 前端静态文件 dist/
  -> /api 反向代理到 FastAPI
  -> MySQL
  -> 持久化 uploads/
```

## 上线前检查

- 生产环境 `.env` 已配置真实 `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_MODEL`。
- `CORS_ALLOW_ORIGINS` 只包含正式域名。
- 前端已用正式 `VITE_API_BASE_URL` 重新构建。
- 首位管理员已通过引导密钥创建，密码强度足够，引导密钥已轮换或删除。
- 普通用户无法访问其他用户存档。
- 管理员后台只能管理员访问。
- 注册、登录、验证码、创建存档、推进剧情、导入导出、头像上传都已在服务器上冒烟测试。
- 确认 GitHub 仓库里没有真实 `.env`、数据库、上传文件或 API Key。
