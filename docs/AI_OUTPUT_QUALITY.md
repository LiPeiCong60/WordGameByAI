# AI 输出质量与诊断

本项目过去偶发的“不如人意”不是单一提示词问题，而是格式链路和内容约束同时存在缺口。

## 已修复的格式链路

- 上下文不再按字符串硬截断；压缩后始终是有效 JSON，并优先保留当前世界、主角、检索记忆和最近回合。
- RAG 使用稳定哈希向量，服务重启后检索结果不会因 Python 随机哈希而漂移；旧向量会按版本惰性重建。
- 可见叙事上下文会移除角色隐藏目标，避免秘密提前泄露。
- 结构化响应解析支持 Markdown 代码块、BOM、尾逗号和混合文本中的首个完整 JSON 对象。
- 内部状态标签会从玩家可见文本中清除。

## 已加强的内容约束

叙事提示词现在要求每轮产生可观察的因果推进，响应玩家的具体行动，避免复述输入、空泛总结、替玩家做关键选择和无铺垫的设定跳跃。结构化分析 Agent 使用较低温度，以减少状态补丁和检查结果漂移。

## 如何读取诊断

每个回合的 `checker_result.output_quality` 会给出 `score`、`likely_cause`、`issues` 和基础指标：

- `format_or_parser`：检测到 JSON 包装、内部标签泄露、格式漂移等，优先检查模型兼容性和输出解析。
- `prompt_context_or_model`：格式正常但内容过短、重复、缺乏推进或偏离上下文，优先调整模型等级、游戏设定和提示词。
- `none`：未命中确定性规则，不等于文学质量完美，只表示没有发现当前规则覆盖的问题。

“快速”模式会跳过部分前置角色推演并异步更新状态，延迟更低；对复杂群像、强设定一致性或重要剧情节点，建议选择“精细”模式。
