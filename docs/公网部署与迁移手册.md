# 公网部署与迁移手册

这份文档用于把 WordGameByAI 部署到公网服务器，也用于以后更换服务器时按步骤迁移。文档里的密码、域名、API Key 都是占位示例，真实值只应写在服务器私有配置中，不要提交到 GitHub。

## 目标架构

```text
用户浏览器
  -> 域名 / 公网 IP
  -> Nginx
  -> frontend/dist 静态文件
  -> /api 反向代理到 127.0.0.1:8010
  -> FastAPI 后端
  -> MySQL
  -> backend/uploads
  -> backend/.env 和 backend/model_configs.json
```

推荐约定：

- 项目目录：`/opt/wordgame`
- 后端服务：`wordgame-backend`
- 后端监听：`127.0.0.1:8010`
- 前端构建目录：`/opt/wordgame/frontend/dist`
- MySQL 数据库：`wordgame`
- MySQL 用户：`wordgame`

## 一、服务器准备

建议最低配置：

- 2 vCPU / 4 GiB 内存
- 40 GiB 系统盘
- Linux 64 位系统
- 公网安全组开放 `80`、`443`，SSH 端口只给自己的 IP 开放更好
- 不要把 `8010`、`3306` 直接开放到公网

安装基础工具：

```bash
sudo dnf update -y
sudo dnf install -y git nginx mysql-server python3 python3-pip python3-devel gcc
```

安装 Node.js 20：

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo dnf install -y nodejs
node -v
npm -v
```

启动 Nginx 和 MySQL：

```bash
sudo systemctl enable --now nginx
sudo systemctl enable --now mysqld
```

如果系统提供的是 MariaDB，也可以使用 MariaDB，项目通过 `PyMySQL` 连接，部署方式基本一致。

## 二、创建 MySQL 数据库

进入 MySQL 控制台。注意下面这些 SQL 必须在 `mysql` 控制台里执行，不是在 Linux shell 里直接执行。

```bash
sudo mysql
```

执行：

```sql
CREATE DATABASE wordgame CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'wordgame'@'127.0.0.1' IDENTIFIED BY '换成强密码';
GRANT ALL PRIVILEGES ON wordgame.* TO 'wordgame'@'127.0.0.1';
FLUSH PRIVILEGES;
EXIT;
```

验证连接：

```bash
mysql -uwordgame -p -h127.0.0.1 wordgame -e "SHOW TABLES;"
```

如果密码里包含 `@`、`:`、`/`、`#` 等特殊字符，写入 `DATABASE_URL` 时需要 URL encode。最省心的方式是先用只包含字母、数字和下划线的强密码。

## 三、拉取项目代码

```bash
cd /opt
sudo git clone https://github.com/LiPeiCong60/WordGameByAI.git wordgame
sudo chown -R root:root /opt/wordgame
cd /opt/wordgame
```

如果服务器直接访问 GitHub 不稳定，可以临时使用代理地址：

```bash
git clone --depth 1 https://gh-proxy.com/https://github.com/LiPeiCong60/WordGameByAI.git wordgame
```

如果是迁移旧服务器，建议记录旧服务器代码版本并在新服务器 checkout 到同一个提交：

```bash
cd /opt/wordgame
git rev-parse HEAD
git checkout <旧服务器提交哈希>
```

## 四、配置后端

安装 Python 依赖：

```bash
cd /opt/wordgame
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

创建后端环境变量文件：

```bash
cp backend/.env.example backend/.env
chmod 600 backend/.env
```

编辑 `backend/.env`：

```env
OPENAI_API_KEY=your_api_key_here
OPENAI_BASE_URL=https://api.deepseek.com
OPENAI_MODEL=deepseek-chat
MODEL_CONFIG_FILE=/opt/wordgame/backend/model_configs.json
DATABASE_URL=mysql+pymysql://wordgame:换成强密码@127.0.0.1:3306/wordgame?charset=utf8mb4
CORS_ALLOW_ORIGINS=http://你的域名,https://你的域名
ADMIN_BOOTSTRAP_TOKEN=替换成长随机字符串
ALLOW_FIRST_USER_ADMIN=false
ACCESS_TOKEN_TTL_MINUTES=60
REFRESH_TOKEN_TTL_DAYS=14
NON_MEMBER_DAILY_MESSAGE_LIMIT=20
MESSAGE_RATE_LIMIT_WINDOW_SECONDS=60
MESSAGE_RATE_LIMIT_MAX_REQUESTS=10
```

说明：

- `OPENAI_API_KEY` 是兜底模型配置。后台模型池里配置的 API Key 会写入 `MODEL_CONFIG_FILE` 指向的服务器私有文件。
- `backend/model_configs.json` 不要提交到 GitHub。前端接口只会看到 `has_api_key`，不会拿到 API Key 明文。
- `CORS_ALLOW_ORIGINS` 上线后要改成正式域名，不要长期使用 `*`。
- 如果暂时只用公网 IP，没有域名，可以先写 `http://公网IP`。
- `ADMIN_BOOTSTRAP_TOKEN` 只用于创建首位管理员；成功后应轮换或删除，公开注册用户不会自动成为管理员。

## 五、创建 systemd 后端服务

写入服务文件：

```bash
sudo tee /etc/systemd/system/wordgame-backend.service >/dev/null <<'EOF'
[Unit]
Description=WordGameByAI Backend
After=network.target mysqld.service mariadb.service

[Service]
WorkingDirectory=/opt/wordgame/backend
EnvironmentFile=/opt/wordgame/backend/.env
ExecStart=/opt/wordgame/.venv/bin/uvicorn main:app --host 127.0.0.1 --port 8010
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF
```

启动后端：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now wordgame-backend
sudo systemctl status wordgame-backend
```

查看日志：

```bash
sudo journalctl -u wordgame-backend -n 100 --no-pager
sudo journalctl -u wordgame-backend -f
```

本机健康检查：

```bash
curl http://127.0.0.1:8010/api/health
```

## 六、构建前端

```bash
cd /opt/wordgame/frontend
npm install
VITE_API_BASE_URL=http://你的域名或公网IP/api npm run build
```

如果已经配置 HTTPS，使用：

```bash
VITE_API_BASE_URL=https://你的域名/api npm run build
```

每次修改前端 API 地址后，都需要重新执行 `npm run build`。

## 七、配置 Nginx

创建 Nginx 配置：

```bash
sudo tee /etc/nginx/conf.d/wordgame.conf >/dev/null <<'EOF'
server {
    listen 80;
    server_name 你的域名或公网IP;

    root /opt/wordgame/frontend/dist;
    index index.html;

    client_max_body_size 10m;

    location /api/ {
        proxy_pass http://127.0.0.1:8010/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }

    location /uploads/ {
        alias /opt/wordgame/backend/uploads/;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
EOF
```

检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

公网检查：

```bash
curl http://你的域名或公网IP/api/health
```

## 八、HTTPS

正式公网建议启用 HTTPS。可以用云厂商证书、宝塔面板证书、Nginx 手动证书，或 Certbot。

使用 Certbot 的常见方式：

```bash
sudo dnf install -y certbot python3-certbot-nginx
sudo certbot --nginx -d 你的域名
```

启用 HTTPS 后要同步改两处：

- `backend/.env` 的 `CORS_ALLOW_ORIGINS=https://你的域名`
- 前端重新构建：`VITE_API_BASE_URL=https://你的域名/api npm run build`

## 九、首次上线检查

1. 打开 `http://你的域名或公网IP`。
2. 注册首个管理员账号，并在注册页填写服务器上的 `ADMIN_BOOTSTRAP_TOKEN`；创建成功后轮换或删除该密钥。
3. 登录后进入后台，检查用户列表、存档列表、模型配置。
4. 在后台模型配置里添加模型池和模型等级，API Key 只填在后台，不会暴露给前端。
5. 创建模板或存档，推进一轮剧情。
6. 检查后端日志没有异常：

```bash
sudo journalctl -u wordgame-backend -n 100 --no-pager
```

## 十、日常更新部署

```bash
cd /opt/wordgame
git pull

source .venv/bin/activate
pip install -r requirements.txt

sudo systemctl restart wordgame-backend

cd frontend
VITE_API_BASE_URL=https://你的域名/api npm run build

sudo nginx -t
sudo systemctl reload nginx
```

如果还没上 HTTPS，把 `https://你的域名/api` 换成 `http://公网IP/api`。

## 十一、备份

建议至少备份三类内容：

- MySQL 数据库
- 后端私有配置：`/opt/wordgame/backend/.env`
- 模型私有配置：`/opt/wordgame/backend/model_configs.json`
- 用户上传：`/opt/wordgame/backend/uploads/`

创建备份目录：

```bash
sudo mkdir -p /opt/backups/wordgame
sudo chmod 700 /opt/backups/wordgame
```

备份数据库：

```bash
mysqldump -uwordgame -p --single-transaction --routines --triggers wordgame \
  > /opt/backups/wordgame/wordgame_$(date +%F_%H%M%S).sql
```

备份私有文件：

```bash
sudo tar -czf /opt/backups/wordgame/wordgame_private_$(date +%F_%H%M%S).tar.gz \
  /opt/wordgame/backend/.env \
  /opt/wordgame/backend/model_configs.json \
  /opt/wordgame/backend/uploads
```

备份文件包含密钥，不能公开上传，建议加密保存或放在受限存储里。

## 十二、迁移服务器流程

旧服务器：

```bash
cd /opt/wordgame
git rev-parse HEAD

mysqldump -uwordgame -p --single-transaction --routines --triggers wordgame > /tmp/wordgame.sql

sudo tar -czf /tmp/wordgame_private.tar.gz \
  /opt/wordgame/backend/.env \
  /opt/wordgame/backend/model_configs.json \
  /opt/wordgame/backend/uploads
```

把 `/tmp/wordgame.sql` 和 `/tmp/wordgame_private.tar.gz` 安全传到新服务器。

新服务器：

```bash
cd /opt
git clone https://github.com/LiPeiCong60/WordGameByAI.git wordgame
cd /opt/wordgame
git checkout <旧服务器提交哈希>

python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

创建数据库和用户后，恢复数据库：

```bash
mysql -uwordgame -p -h127.0.0.1 wordgame < /tmp/wordgame.sql
```

恢复私有文件：

```bash
sudo tar -xzf /tmp/wordgame_private.tar.gz -C /
sudo chmod 600 /opt/wordgame/backend/.env
sudo chmod 600 /opt/wordgame/backend/model_configs.json
```

然后按本文的 systemd、前端构建和 Nginx 步骤启动新服务器。

迁移完成后检查：

```bash
curl http://127.0.0.1:8010/api/health
curl http://你的新域名或公网IP/api/health
sudo journalctl -u wordgame-backend -n 100 --no-pager
```

确认登录、后台、模板、存档、剧情推进、模型调用都正常后，再切换域名 DNS。

## 十三、安全清单

- `.env`、`model_configs.json`、数据库文件、上传文件不要提交到 GitHub。
- 后台模型配置接口不能返回 API Key 明文，只允许返回是否已配置。
- 生产环境必须用强密码，不要使用示例密码。
- MySQL 只监听本机或内网，不要开放公网 `3306`。
- 后端只监听 `127.0.0.1:8010`，由 Nginx 反向代理。
- SSH 端口不要对全网开放，至少使用强密码，最好改用密钥登录。
- 登录、注册、验证码接口建议加 Nginx 层 IP 限流。
- 正式域名启用 HTTPS 后，再在后台输入真实模型 API Key。
- 备份文件包含密钥和用户数据，要私密保存。

## 十四、常见问题

### SQL 语句提示 command not found

你把 SQL 写在 Linux shell 里了。先执行：

```bash
sudo mysql
```

再执行 `CREATE DATABASE`、`CREATE USER`、`GRANT`。

### 端口 8010 已被占用

通常是 systemd 服务已经在运行，你又手动启动了一次 uvicorn。检查：

```bash
sudo systemctl status wordgame-backend
sudo ss -lntp | grep 8010
```

需要重启时用：

```bash
sudo systemctl restart wordgame-backend
```

### 前端打开正常，但 API 报错或 502

检查后端：

```bash
sudo systemctl status wordgame-backend
sudo journalctl -u wordgame-backend -n 100 --no-pager
curl http://127.0.0.1:8010/api/health
```

检查 Nginx：

```bash
sudo nginx -t
sudo tail -n 100 /var/log/nginx/error.log
```

### 浏览器提示跨域

检查 `backend/.env`：

```env
CORS_ALLOW_ORIGINS=https://你的域名
```

修改后重启后端：

```bash
sudo systemctl restart wordgame-backend
```

### 模型调用失败

检查：

- `backend/.env` 里的兜底模型配置是否正确。
- 后台模型池是否填了 API Key。
- 用户是否分配了正确模型等级。
- 模型等级是否给对应 Agent 指定了模型。
- `MODEL_CONFIG_FILE` 指向的文件是否存在且后端有权限读取。

查看日志：

```bash
sudo journalctl -u wordgame-backend -f
```

### 修改后前端没有变化

前端是静态构建产物，需要重新构建：

```bash
cd /opt/wordgame/frontend
VITE_API_BASE_URL=https://你的域名/api npm run build
sudo systemctl reload nginx
```
